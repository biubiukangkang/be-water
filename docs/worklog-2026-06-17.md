## 二、遇到的问题与解决方案

### 1. 路径含中文导致编译失败

**问题**: Android Gradle Plugin 拒绝在含中文的路径下编译
**解决**: `gradle.properties` 添加 `android.overridePathCheck=true`
**经验**: Windows 上 Android 开发路径不能含中文，这是 AGP 硬性检查

### 2. `combine()` 函数 7 参数重载不识别

**问题**: `kotlinx.coroutines.flow.combine(flow1..flow7)` 编译报类型推断失败
**解决**: 改用链式 `.combine()` 逐个合并，每次合并一个 Flow → `listOf<Any>()` + `l + element`
**经验**: Kotlin 1.9.22 + kotlinx-coroutines 1.7.1 对 `combine` 多参数重载存在编译器 bug

### 3. HTML blur 效果在 Compose 中不可用

**问题**: 网页版的 `backdrop-filter: blur(16px)` 在 Android Compose 中没有直接等价物
**解决**: 提高半透明颜色的 alpha 值（2-3x）来补偿视觉深度
**经验**: CSS 的模糊效果不能照搬到 Compose，需要用更高透明度+边框模拟玻璃态

### 4. Compose shadow() 渲染多边形

**问题**: `Modifier.shadow(8.dp, CircleShape)` 在某些设备上渲染为六边形/矩形阴影
**解决**: 全局删除 `Modifier.shadow()`，所有按钮只用 `background()` + `border()` 
**经验**: Android GPU 的 Outline Provider 对圆形 shadow 的近似不可靠，小按钮尤其明显

### 5. ModalBottomSheet 颜色不跟随

**问题**: 设置/统计面板的背景和文字颜色与主题不一致
**解决**: 显式传入 `containerColor` 和 `contentColor` 给 `ModalBottomSheet`
**经验**: Material3 的 ModalBottomSheet 默认使用主题颜色，当页面背景与主题背景不同时需要手动覆盖

### 6. 顶部栏被系统状态栏遮挡

**问题**: Scaffold 的 topBar 绘制在状态栏下方（被遮住）
**解决**: topBar 的 Row 加上 `Modifier.windowInsetsPadding(WindowInsets.statusBars)`
**经验**: Compose 中 Scaffold 的 topBar 不会自动处理状态栏 inset，需要手动添加

### 7. 颜色透明度在真机与模拟器差异大

**问题**: 在模拟器上看得见的玻璃按钮，真机完全透明
**解决**: 
- 深色模式 `DarkPrimary` alpha: 0.20 → 0.35
- 浅色模式 `LightPrimary` alpha: 0.08 → 0.40（且改为白色基底）
- 按钮边框 alpha: 0.15 → 0.25
- 圆环轨道 alpha: 0.12 → 0.22
**经验**: 模拟器亮度/对比度与真机不同，重要 UI 必须在真机验证
